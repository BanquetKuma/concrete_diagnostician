# 受け入れ基準

## 概要

このドキュメントでは、コンクリート診断士試験対策アプリMVPの受け入れ基準を定義します。各機能・品質要件に対して明確で検証可能な基準を設定し、プロダクトの完成度を客観的に判断できるようにします。

## 受け入れ基準の評価方法

### 評価レベル

- **✅ Pass**: 基準を満たしている
- **❌ Fail**: 基準を満たしていない
- **⚠️ Conditional**: 条件付きで許可（軽微な問題）

### 評価プロセス

1. **自動テスト**: 技術的基準の自動検証
2. **手動テスト**: UI/UX・ビジネスロジックの手動確認
3. **ステークホルダーレビュー**: ビジネス要件の最終確認

## 機能要件の受け入れ基準

### F-01: ユーザー管理機能

#### AC-F01-001: デバイスベースユーザー管理

**Given** アプリを初回起動する
**When** アプリがロードされる  
**Then** 自動的にユニークなユーザーIDが生成される

**検証方法**:

- [ ] UUID形式のユーザーIDが生成される
- [ ] 同一デバイスで再起動してもIDが維持される
- [ ] Expo Secure StoreにIDが永続化される
- [ ] 生成処理が3秒以内に完了する

**合格基準**: 上記4項目すべてPass

#### AC-F01-002: ユーザー情報保存

**Given** ユーザーIDが生成される
**When** 初回データ同期が実行される
**Then** Azure Cosmos DBにユーザードキュメントが作成される

**検証方法**:

- [ ] Cosmos DBにユーザードキュメントが存在する
- [ ] ドキュメント内容がスキーマに準拠している
- [ ] 作成日時・最終アクセス日時が正確に記録される
- [ ] API通信エラー時の適切なハンドリング

**合格基準**: 上記4項目すべてPass

### F-02: ホーム画面機能

#### AC-F02-001: 年度別問題リスト表示

**Given** ユーザーがホーム画面にアクセスする
**When** 画面が読み込まれる
**Then** 年度別の問題リストが表示される

**検証方法**:

- [ ] 年度リスト（2024年度、2023年度等）が表示される
- [ ] 各年度の表示が視覚的に区別できる
- [ ] タップ可能であることが明確にわかる
- [ ] データ読み込み中は適切なローディング表示
- [ ] エラー時は理解しやすいエラーメッセージ

**合格基準**: 上記5項目すべてPass

#### AC-F02-002: 学習進捗の可視化

**Given** 学習履歴データが存在する
**When** ホーム画面が表示される
**Then** 年度別の進捗状況が表示される

**検証方法**:

- [ ] 「15/50」形式で進捗数値が表示される
- [ ] 進捗率に応じたプログレスバーが表示される
- [ ] 未着手の年度は「0/50」と表示される
- [ ] 進捗が更新されるとリアルタイムに反映される

**合格基準**: 上記4項目すべてPass

### F-03: 問題演習機能（コア機能）

#### AC-F03-001: 問題データ表示

**Given** 問題演習画面にアクセスする
**When** 問題データが読み込まれる
**Then** 問題内容が適切に表示される

**検証方法**:

- [ ] 問題番号が画面上部に表示される
- [ ] 問題文が読みやすく表示される
- [ ] 4つの選択肢がボタンとして配置される
- [ ] 文字サイズが適切で読みやすい
- [ ] 画面サイズに応じてレスポンシブに調整される

**合格基準**: 上記5項目すべてPass

#### AC-F03-002: 解答選択・正誤判定

**Given** 問題が表示されている
**When** ユーザーが選択肢をタップする
**Then** 適切な正誤判定とフィードバックが提供される

**検証方法**:

- [ ] タップした選択肢の色が変わる（選択状態表示）
- [ ] 他の選択肢がタップできなくなる
- [ ] 正解の選択肢が緑色でハイライトされる
- [ ] 不正解の選択肢が赤色でハイライトされる
- [ ] 判定処理が1秒以内に完了する

**合格基準**: 上記5項目すべてPass

#### AC-F03-003: 解説表示

**Given** 解答が完了している
**When** 正誤判定が表示される
**Then** 詳細な解説が表示される

**検証方法**:

- [ ] 画面下部に解説エリアが表示される
- [ ] 解説文がスクロール可能である
- [ ] 文字が読みやすいサイズとコントラスト
- [ ] 解説内容が問題に関連している
- [ ] 専門用語の説明が含まれている

**合格基準**: 上記5項目すべてPass

#### AC-F03-004: 学習履歴保存

**Given** ユーザーが問題に解答する
**When** 解答が完了する
**Then** 学習履歴が確実に保存される

**検証方法**:

- [ ] 解答結果（正解・不正解）が記録される
- [ ] 解答時刻が正確に記録される
- [ ] 問題IDとユーザーIDが紐づけられる
- [ ] オフライン時は一時保存される
- [ ] オンライン復帰時に同期される

**合格基準**: 上記5項目すべてPass

#### AC-F03-005: 問題間ナビゲーション

**Given** 問題演習画面で解答が完了している
**When** ナビゲーションボタンをタップする
**Then** 適切な画面遷移が実行される

**検証方法**:

- [ ] 「次の問題へ」ボタンで次問題に遷移
- [ ] 「問題一覧へ戻る」ボタンでホーム画面に戻る
- [ ] 最後の問題では「完了」ボタンに変更される
- [ ] 画面遷移が1秒以内に完了する
- [ ] ナビゲーション状態が適切に管理される

**合格基準**: 上記5項目すべてPass

### F-04: コンテンツ管理・品質保証

#### AC-F04-001: AI解説品質

**Given** 問題データが存在する
**When** 解説が表示される
**Then** 高品質な解説コンテンツが提供される

**検証方法**:

- [ ] 専門家による監修が完了している
- [ ] 技術的内容に誤りがない
- [ ] 初学者にもわかりやすい表現
- [ ] 200-400文字の適切な長さ
- [ ] 不正解選択肢の説明も含まれている

**合格基準**: 上記5項目すべてPass（専門家承認必須）

## 非機能要件の受け入れ基準

### NF-01: パフォーマンス要件

#### AC-NF01-001: 応答時間

**検証方法**:

- [ ] 画面遷移時間: 0.5秒以内
- [ ] API応答時間: 1.5秒以内
- [ ] アプリ初回起動: 3秒以内
- [ ] 問題読み込み: 2秒以内

**測定方法**: Application Insightsでの実測値
**合格基準**: 上記4項目すべてPass

#### AC-NF01-002: 安定性

**検証方法**:

- [ ] クラッシュ率: 0.1%以下
- [ ] メモリリーク: 検出されない
- [ ] CPUメタリー使用率: 平均50%以下
- [ ] バッテリー消費: 通常アプリレベル

**測定方法**: Xcode Instrumentsでの計測
**合格基準**: 上記4項目すべてPass

### NF-02: セキュリティ要件

#### AC-NF02-001: データ保護

**検証方法**:

- [ ] 機密データがSecure Storeに暗号化保存
- [ ] API通信がHTTPS強制
- [ ] ユーザーIDが推測不可能な形式
- [ ] 個人情報の外部送信なし

**測定方法**: セキュリティスキャンとコードレビュー
**合格基準**: 上記4項目すべてPass

### NF-03: 使いやすさ要件

#### AC-NF03-001: UI/UX品質

**検証方法**:

- [ ] タップターゲットが44pt以上
- [ ] 色のみに依存しない情報伝達
- [ ] 明確な視覚階層とナビゲーション
- [ ] iOS Human Interface Guidelines準拠

**測定方法**: アクセシビリティチェックとユーザビリティテスト
**合格基準**: 上記4項目すべてPass

### NF-04: 互換性要件

#### AC-NF04-001: デバイス対応

**検証方法**:

- [ ] iPhone 12以降で正常動作
- [ ] iOS 14.0以降で動作
- [ ] 各種画面サイズで適切表示
- [ ] ダークモード対応

**測定方法**: 実機テストでの動作確認
**合格基準**: 上記4項目すべてPass
