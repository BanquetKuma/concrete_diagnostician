# マイルストーン計画

## 概要

本ドキュメントでは、コンクリート診断士試験対策アプリMVPの開発における詳細なマイルストーン計画を定義します。

## プロジェクト全体スケジュール

**プロジェクト期間**: 7週間（35営業日）
**開始予定**: 2025年第1四半期
**完了予定**: 2025年第2四半期初旬

## マイルストーン概要

### M0: プロジェクト開始・準備完了

- **期間**: Week 1 (5営業日)
- **ゲート基準**: 開発環境、Azure環境、法的許諾の完全準備
- **リスク**: 著作権許諾取得の遅延

### M1: 基本機能完成

- **期間**: Week 2-3 (8営業日)
- **ゲート基準**: ユーザー管理とホーム画面の動作確認
- **リスク**: Azure統合の複雑性

### M2: コア機能完成

- **期間**: Week 3-6 (15営業日)
- **ゲート基準**: 問題演習の完全動作とAI解説統合
- **リスク**: OpenAI統合とコンテンツ品質

### M3: リリース準備完了

- **期間**: Week 6-7 (7営業日)
- **ゲート基準**: App Store提出可能な品質レベル達成
- **リスク**: App Store審査準備の不備

## 詳細マイルストーン定義

## M0: プロジェクト開始・準備完了

### 期間・スコープ

- **期間**: Day 1-5 (5営業日)
- **担当**: 全チームメンバー
- **並行作業**: 可能な限り並行実行で効率化

### 成功基準（Definition of Done）

- [ ] **法的許諾完了**: JCIから過去問利用の正式許諾書面取得
- [ ] **開発環境構築**: 全開発者のローカル環境でExpo/React Nativeが動作
- [ ] **Azure環境準備**:
  - Azure Cosmos DB インスタンス作成・接続確認
  - Azure Functions デプロイ環境構築
  - Azure OpenAI Service エンドポイント設定
  - Application Insights 設定完了
- [ ] **プロジェクト初期化**:
  - TypeScript/ESLint設定完了
  - 基本ディレクトリ構造作成
  - CI/CD パイプライン基本設定
- [ ] **チーム体制確立**: 役割分担・コミュニケーションルール合意

### 主要タスク

| タスク                    | 担当者       | 所要時間 | 依存関係        |
| ------------------------- | ------------ | -------- | --------------- |
| 著作権許諾取得            | PM           | 継続作業 | なし            |
| Expo プロジェクト初期設定 | Frontend Dev | 2時間    | なし            |
| Azure リソース作成        | Backend Dev  | 6時間    | Azureアカウント |
| 開発ガイドライン作成      | Tech Lead    | 2時間    | なし            |
| デザインシステム設計      | UI/UX        | 4時間    | なし            |

### リスク・対策

- **🔴 高リスク**: 著作権許諾の遅延 → プロジェクト中止リスク
  - **対策**: 許諾取得を最優先、代替コンテンツ戦略を並行検討
- **🟡 中リスク**: Azure環境設定の複雑性 → 開発遅延
  - **対策**: Azure専門家のサポート確保、詳細ドキュメント作成

### マイルストーンゲート

- **Go/No-Go判定**: Day 5 EOD
- **判定基準**: 上記成功基準の100%達成
- **エスカレーション**: 未達成の場合、プロジェクトマネージャーへ即座に報告

## M1: 基本機能完成

### 期間・スコープ

- **期間**: Day 6-13 (8営業日)
- **担当**: 主にフロントエンド開発者
- **フォーカス**: ユーザー管理とホーム画面の実装

### 成功基準（Definition of Done）

  - 年度別問題リスト表示
  - Azure Functionsからのデータ取得
  - ローディング・エラー状態の適切な表示
- [ ] **ナビゲーション**:
  - Expo Routerでの画面遷移実装
  - 基本的なUI/UX動作確認
- [ ] **品質基準**:
  - TypeScript型チェック完全パス
  - ESLint違反ゼロ
  - 基本的な単体テスト実装

### 主要成果物

- ユーザー管理コンポーネント群
- ホーム画面コンポーネント群
- Azure Functions（ユーザー・問題データAPI）
- Zustand状態管理ストア

### デモ・検証内容

- **内部デモ**: Day 11実施
- **検証項目**:
  - アプリ起動からホーム画面表示まで正常動作
  - 年度リストタップで問題一覧への遷移
  - オフライン→オンライン復帰時の動作
  - 複数デバイスでの動作確認

### リスク・対策

- **🟡 中リスク**: Azure Functions統合の複雑性
  - **対策**: シンプルなREST API設計、早期統合テスト実施
- **🟡 中リスク**: Expo Routerの習得コスト
