# ユーザーストーリー一覧

## 概要

このドキュメントでは、コンクリート診断士試験対策アプリMVPのユーザーストーリーを定義します。各ストーリーはAs a [role], I want [feature], so that [benefit]の形式で記述されています。

## ユーザーペルソナ

### プライマリペルソナ：受験者（学習者）

- **役割**: コンクリート診断士試験の受験予定者
- **特徴**: 多忙な社会人、限られた学習時間、効率的な学習を求める
- **課題**: 重い専門書の携帯、スキマ時間の活用困難、進捗把握の難しさ

## エピック1: ユーザー管理機能

### US-001: 初回アプリ起動時のユーザー識別

- **ユーザーストーリー**:
  - As a 受験者, I want アプリ初回起動時に自動的にユーザーIDが生成される, so that 面倒な登録手続きなしにすぐに学習を開始できる
- **ストーリーポイント**: 3
- **優先度**: High
- **受け入れ基準**:
  - アプリ初回起動時にUUIDが自動生成される
  - 生成されたIDがExpo Secure Storeに永続化される
  - ユーザーは登録フローを経ることなく直接ホーム画面に遷移する
- **技術的タスク**:
  - [ ] UUID生成ロジック実装
  - [ ] Secure Store連携実装
  - [ ] 初回起動判定機能

### US-002: ユーザーデータの永続化

- **ユーザーストーリー**:
  - As a 受験者, I want 学習履歴が自動的に保存される, so that アプリを再インストールしても学習進捗を失わない
- **ストーリーポイント**: 5
- **優先度**: High
- **受け入れ基準**:
  - ユーザーIDがAzure Cosmos DBに保存される
  - 学習履歴が問題ごとに記録される
  - アプリ再インストール後も同じデバイスなら履歴が復元される
- **技術的タスク**:
  - [ ] Azure Cosmos DBスキーマ設計
  - [ ] ユーザー作成API実装
  - [ ] データ同期ロジック実装

## エピック2: ホーム画面・ナビゲーション

### US-003: 年度別問題リストの表示

- **ユーザーストーリー**:
  - As a 受験者, I want 過去問を年度別に整理して表示される, so that 学習したい年度の問題を素早く見つけられる
- **ストーリーポイント**: 5
- **優先度**: High
- **受け入れ基準**:
  - ホーム画面に年度リスト（例：2024年度、2023年度）が表示される
  - 各年度をタップすると該当年度の問題一覧に遷移する
  - データが未取得の場合、適切なローディング状態が表示される
- **技術的タスク**:
  - [ ] 年度リストコンポーネント実装
  - [ ] 問題データ取得API連携
  - [ ] ローディング・エラー状態管理

### US-004: 学習進捗の可視化

- **ユーザーストーリー**:
  - As a 受験者, I want 各年度の学習進捗が一目でわかる, so that どの年度の学習が完了し、どこが未着手かを把握できる
- **ストーリーポイント**: 3
- **優先度**: Medium
- **受け入れ基準**:
  - 年度リストの横に「15/50」形式で進捗が表示される
  - 進捗率に応じて視覚的にわかりやすい表示（プログレスバー等）
  - リアルタイムで進捗が更新される
- **技術的タスク**:
  - [ ] 進捗計算ロジック実装
  - [ ] プログレスバーコンポーネント
  - [ ] リアルタイム更新機能

### US-005: 直感的なナビゲーション

- **ユーザーストーリー**:
  - As a 受験者, I want アプリ内の移動が直感的でわかりやすい, so that 学習に集中でき、迷うことなく目的の画面にアクセスできる
- **ストーリーポイント**: 2
- **優先度**: Medium
- **受け入れ基準**:
  - 戻るボタンが適切に配置されている
  - ブレッドクラム等で現在位置が分かる
  - タップ可能な要素が明確に識別できる
- **技術的タスク**:
  - [ ] Expo Routerナビゲーション実装
  - [ ] 共通ヘッダーコンポーネント
  - [ ] ナビゲーション状態管理

## エピック3: 問題演習機能（コア機能）

### US-006: 問題内容の表示

- **ユーザーストーリー**:
  - As a 受験者, I want 問題文と選択肢が読みやすく表示される, so that 内容を正確に理解して解答できる
- **ストーリーポイント**: 3
- **優先度**: High
- **受け入れ基準**:
  - 問題番号、問題文が画面上部に表示される
  - 4つの選択肢がボタンとして表示される
  - 文字サイズが適切で読みやすい
  - 画面サイズに応じてレスポンシブに表示される
- **技術的タスク**:
  - [ ] 問題表示コンポーネント実装
  - [ ] レスポンシブデザイン対応
  - [ ] フォントサイズ・行間調整

### US-007: 選択肢の選択と解答

- **ユーザーストーリー**:
  - As a 受験者, I want 選択肢をタップして解答を選択できる, so that 直感的な操作で問題に回答できる
- **ストーリーポイント**: 3
- **優先度**: High
- **受け入れ基準**:
  - 選択肢ボタンをタップすると選択状態になる
  - 選択済みボタンのスタイルが変更される（背景色等）
  - 一度選択すると他の選択肢はタップできなくなる
  - 選択後は自動的に正誤判定が実行される
- **技術的タスク**:
  - [ ] 選択肢ボタンコンポーネント
  - [ ] 選択状態管理ロジック
  - [ ] タップ無効化機能

### US-008: 即座の正誤判定とフィードバック

- **ユーザーストーリー**:
  - As a 受験者, I want 解答後すぐに正解・不正解がわかる, so that 学習効果を最大化し、すぐに次の行動に移れる
- **ストーリーポイント**: 3
- **優先度**: High
- **受け入れ基準**:
  - 選択肢選択後、即座に正誤判定が実行される
  - 正解の選択肢は緑色でハイライト表示される
  - 不正解の選択肢は赤色でハイライト表示される
  - 判定結果に応じたアニメーション効果がある
- **技術的タスク**:
  - [ ] 正誤判定ロジック実装
  - [ ] ハイライト表示機能
  - [ ] フィードバックアニメーション

### US-009: 詳細な解説の表示

- **ユーザーストーリー**:
  - As a 受験者, I want 解答後に詳しい解説を読める, so that なぜその答えが正解なのか理解でき、知識が定着する
- **ストーリーポイント**: 4
- **優先度**: High
- **受け入れ基準**:
  - 正誤判定後、画面下部に解説エリアが表示される
  - 解説文がスクロール可能で、長文にも対応している
  - 解説内容はAI生成され、専門家による監修済み
  - 解説の表示・非表示を切り替えられる
- **技術的タスク**:
  - [ ] 解説表示コンポーネント
  - [ ] スクロール機能実装
  - [ ] 折りたたみ表示機能

### US-010: 学習履歴の自動保存

- **ユーザーストーリー**:
  - As a 受験者, I want 解答結果が自動的に記録される, so that 後で学習状況を振り返ることができる
- **ストーリーポイント**: 5
- **優先度**: High
- **受け入れ基準**:
  - 各問題の解答結果（正解・不正解・解答時刻）が保存される
  - オフライン時も一時保存され、オンライン復帰時に同期される
  - 解答履歴をもとに進捗が正確に計算される
- **技術的タスク**:
  - [ ] 履歴保存API実装
  - [ ] オフライン対応
  - [ ] 進捗計算ロジック

### US-011: 問題間のスムーズな移動

- **ユーザーストーリー**:
  - As a 受験者, I want 問題を連続して解いていける, so that 集中を切らすことなく効率的に学習できる
- **ストーリーポイント**: 2
- **優先度**: Medium
- **受け入れ基準**:
  - 「次の問題へ」ボタンで次の問題に遷移する
  - 「問題一覧へ戻る」ボタンで一覧画面に戻れる
  - 最後の問題では「完了」表示に変わる
- **技術的タスク**:
  - [ ] ナビゲーションボタン実装
  - [ ] 問題順序管理
  - [ ] 完了状態の判定

## エピック4: コンテンツ管理・品質保証

### US-012: 高品質な解説コンテンツ

- **ユーザーストーリー**:
  - As a 受験者, I want 専門的で正確な解説を読める, so that 信頼できる情報で効果的に学習できる
- **ストーリーポイント**: 8
- **優先度**: High
- **受け入れ基準**:
  - AI生成された解説が専門家により監修されている
  - 解説内容に誤りがない
  - 初学者にもわかりやすい表現で書かれている
- **技術的タスク**:
  - [ ] Azure OpenAI Service連携
  - [ ] プロンプト最適化
  - [ ] 監修ワークフロー構築

### US-013: 安定したアプリ動作

- **ユーザーストーリー**:
  - As a 受験者, I want アプリがクラッシュせず安定して動作する, so that 学習を中断されることなく集中できる
- **ストーリーポイント**: 5
- **優先度**: High
- **受け入れ基準**:
  - クラッシュ率が0.1%以下
  - 応答時間が1.5秒以内
  - メモリリークがない
- **技術的タスク**:
  - [ ] エラーハンドリング実装
  - [ ] パフォーマンス最適化
  - [ ] メモリ管理改善
