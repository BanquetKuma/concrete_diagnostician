# UI Debug Skill

UIの表示問題を体系的にデバッグするためのスキル。

## デバッグの原則

**表示されているものだけを信じない。必ず元データを確認する。**

## デバッグ手順

### Step 1: 問題の特定

ユーザーから以下を確認：
- どの画面で問題が発生しているか
- 期待される表示と実際の表示の違い
- スクリーンショットがあれば確認

### Step 2: データソースの確認

**最重要ステップ**: 実際のAPIレスポンスを取得して確認する

```bash
# APIエンドポイントを直接叩いてデータ形式を確認
curl -s "APIのURL" | head -c 2000
```

確認ポイント：
- データの実際のフォーマット
- 改行文字（`\n`）の有無
- 特殊文字の形式

### Step 3: 文字エンコーディングの確認

よくある問題：
| 期待 | 実際 | 問題 |
|------|------|------|
| `[正解]` | `【正解】` | 半角括弧 vs 全角括弧 |
| `a:` | `ａ：` | 半角英数 vs 全角英数 |
| `。` | `.` | 句点の違い |
| `\n` | なし | 改行の有無 |

正規表現で両方に対応する例：
```typescript
// 全角・半角両対応
/(?=【正解】|【誤り】|\[正解\]|\[誤り\])/
/[a-dａ-ｄ][：:]/  // コロンも両対応
```

### Step 4: コンポーネントのコード確認

該当するReact Nativeコンポーネントを確認：
- データの取得方法
- データの変換・加工処理
- 表示ロジック

### Step 5: スタイリングの確認

React Nativeでよくあるスタイリング問題：

**問題1: 条件付きスタイルが効かない**
```typescript
// 問題のあるコード
style={[styles.text, index > 0 && styles.margin]}

// 解決策: 明示的にnullを返す
style={[styles.text, index > 0 ? styles.margin : null]}

// または: Viewでラップしてインラインスタイル
<View style={index > 0 ? { marginTop: 20 } : undefined}>
  <Text>{content}</Text>
</View>
```

**問題2: marginTopが効かない**
- 親コンポーネントのレイアウトを確認
- `ThemedView`でラップしてインラインスタイルを適用

### Step 6: デバッグ用ログの追加

一時的にconsole.logを追加してデータを確認：
```typescript
console.log('Raw data:', JSON.stringify(data, null, 2));
console.log('Processed parts:', parts);
```

## チェックリスト

- [ ] APIレスポンスの実データを確認したか
- [ ] 文字エンコーディング（全角/半角）を確認したか
- [ ] 改行文字の有無を確認したか
- [ ] 正規表現が実データに対応しているか
- [ ] スタイルが正しく適用されているか
- [ ] 条件付きスタイルの書き方は正しいか

## よくある原因と解決策

| 症状 | 原因 | 解決策 |
|------|------|--------|
| テキストが分割されない | 正規表現が実データにマッチしない | APIデータを確認し正規表現を修正 |
| 間隔が表示されない | marginTopが効いていない | Viewでラップしてインラインスタイル適用 |
| 一部のデータだけ問題 | データフォーマットの不統一 | 複数パターンに対応する処理を追加 |
| 文字化け | エンコーディング問題 | UTF-8で統一、全角/半角両対応 |

## 参考: 今回の事例

### 問題
解説文の`[正解]`と`[誤り]`の間に間隔が表示されない

### 発見プロセス
1. スクリーンショットでは`[正解]`と`[誤り]`に見えた
2. **APIを直接確認** → 実際は`【正解】`と`【誤り】`（全角括弧）
3. さらにデータには既に`\n`改行が含まれていた
4. 正規表現を全角括弧対応に修正
5. スタイリングも`ThemedView`ラップ + インラインスタイルで確実に適用

### 教訓
**画面表示からの推測ではなく、実データを確認することが最も重要**
