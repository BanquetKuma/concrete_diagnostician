### 1.6 既存サブスクリプターの移行（7名）

#### 移行方針

既存のサブスクリプション購読者は**自動的に買い切り済みとして扱い**、追加購入不要でPro権限を維持する。

| 対象 | 対応 |
|------|------|
| 既存月額購読者（7名） | RevenueCatで旧Product (`...monthly`) が `pro` エンタイトルメントに紐付けられているため、権限は自動継続。コードの `isProMember` 判定は `entitlements.active['pro']` を確認するだけなので、プロダクトの種類を問わない。**既存ユーザーは何もせずにPro会員のまま利用可能** |
| 既存購読者のサブスク自動更新 | Appleの仕組み上、サブスクは自動更新が続く。ユーザーが自ら解約する必要があるが、買い切りに移行済みのため解約しても `pro` 権限は維持される（RevenueCatで両プロダクトが `pro` に紐付いているため） |
| 新規ユーザー | `lifetime` パッケージのみが表示され、買い切りプランのみ購入可能 |

> **重要**: 既存サブスクユーザーには「サブスクリプションを解約しても問題ない」旨をアナウンスする必要あり（下記 1.6.2 参照）

#### 1.6.1 リストア機能の動作

- `restorePurchases()` は旧サブスク・新買い切りの両方を復元する
- RevenueCatが自動的にユーザーの全購入履歴を照合し、`pro` エンタイトルメントの状態を更新する

#### 1.6.2 アプリ内アナウンス（実装が必要）

サブスクから買い切りへの移行をユーザーに告知するモーダルダイアログをアプリに追加する。

**表示形式**: モーダルダイアログ（画面中央にポップアップ、「OK」ボタンで閉じる）
**表示対象**: 全ユーザー（Pro会員・無料会員問わず）
**表示条件**: アプリ更新後の初回起動時（`AsyncStorage` でフラグ管理）
**トーン**: 丁寧・ビジネス的

## Phase 4: チャットbot導入

### 4.1 概要

城岸さんのフィードバック:
> 「チャットbotはあると便利」

AI を活用した質疑応答機能を追加し、学習中の疑問をリアルタイムで解消できるようにする。

### 4.2 技術方針

#### バックエンド

| 選択肢 | メリット | デメリット |
|--------|---------|-----------|
| Azure OpenAI Service (GPT-4) | 既にAzure環境あり、日本リージョン利用可 | コストが高い |
| Cloudflare Workers AI | 既存Workers環境と統合、低コスト | モデル選択肢が限定的 |
| OpenAI API (直接) | 最新モデルが使える | Azure以外の外部依存 |

> **推奨**: Azure OpenAI Service — 既に解説生成で使用実績があり、データの日本国内保存が可能

#### RAG (Retrieval-Augmented Generation)

- 教科書PDFをベクトル化して検索可能にする（既にGemini File Search APIでRAG基盤あり）
- ユーザーの質問に対して、関連する教科書の該当箇所を参照して回答

#### フロントエンド実装

- 新しいタブまたは問題詳細画面内にチャットUIを追加
