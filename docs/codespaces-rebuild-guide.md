# GitHub Codespaces: コンテナ再構築（Rebuild）ガイド

Codespacesの設定ファイルを書き換えた場合、単に再起動するだけでは変更が反映されません。
**「Rebuild（作り直し）」** を行うことで、新しい設定に基づいた環境が再作成されます。

## 1. Rebuildが必要なタイミング

以下のファイルを変更した時は、必ずRebuildを実行してください。

### `devcontainer.json` を変更した時

- VS Codeの拡張機能（Extensions）を追加・削除した
- `postCreateCommand`（起動時のインストールコマンド）を修正した
- ポート設定や環境変数を変更した

### `Dockerfile` を変更した時

- OSのライブラリやツールを追加した

### 環境の不具合を解消したい時

- 環境が汚れて挙動がおかしい場合、Rebuildすることで「初期状態（クリーンな状態）」に戻せます。

## 2. Rebuildの実行方法

以下の2つの方法があります。**方法A（ボタン操作）が最も簡単で確実です。**

### 方法A：画面左下のボタンから実行（推奨）

1. VS Codeの画面左下にある **緑色（または青色）の「><」アイコン** をクリックします。
2. 画面上部にメニューが表示されます。
3. リストの中から **`Rebuild Container`** （または「コンテナを再構築する」）を選択します。

### 方法B：コマンドパレットから実行

1. `F1` キー または `Ctrl + Shift + P` (Mac: `Cmd + Shift + P`) を押してコマンドパレットを開きます。
2. 検索ボックスに以下のキーワードを入力します。
   - `rebuild`
   - `再構築`
   - `container`
3. 候補に出てくる **`Codespaces: Rebuild Container`** をクリックします。

## 3. 注意点

### データについて

- `/workspaces/` フォルダ内（リポジトリのソースコード）のデータは **消えずに保持されます**。
- それ以外の場所（ホームディレクトリなど）に一時的に作成したファイルや、手動でインストールしたツールは **リセット（削除）されます**。

### 所要時間

- コンテナをゼロから作り直すため、完了まで数分かかります（`postCreateCommand` も再実行されます）。
