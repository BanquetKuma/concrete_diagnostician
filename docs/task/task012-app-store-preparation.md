## 📚 学習ポイント：App Store での「見た目」の重要性

### なぜアセットが重要か？

```
【ユーザーの意思決定プロセス】

検索結果 → アイコンが目に入る（0.5秒）
   ↓
気になる → タップしてページを見る（3秒）
   ↓
スクリーンショット → 「使えそう」と判断（5秒）
   ↓
説明文を読む → 「これが必要」と確信（10秒）
   ↓
ダウンロードボタンをタップ

【衝撃の事実】
- ユーザーの60%は説明文を読まずに判断
- スクリーンショットで「雰囲気」を把握
- アイコンはブランドの顔

→ アセットの質がダウンロード数を左右する！
```

### App Store Optimization（ASO）とは

```
【ASO = App Store 版 SEO】

検索で上位に表示される + ページ訪問者がダウンロードする

【ASO の要素】
1. キーワード最適化（検索順位に影響）
   - アプリ名、サブタイトル、キーワードフィールド

2. 視覚最適化（コンバージョン率に影響）
   - アイコン、スクリーンショット、動画

3. 評価・レビュー
   - 星の数、レビュー内容

【MVPでの優先順位】
1位: アイコン（第一印象）
2位: スクリーンショット（機能理解）
3位: 説明文（詳細理解）
```

## 7.1 アプリアイコン

### 必要サイズ

| サイズ | 用途 | ファイル名 |
|--------|------|-----------|
| 1024x1024 | App Store | icon-1024.png |
| 180x180 | iPhone (@3x) | icon-180.png |
| 167x167 | iPad Pro (@2x) | icon-167.png |
| 152x152 | iPad (@2x) | icon-152.png |
| 120x120 | iPhone (@2x) | icon-120.png |

### デザイン要件

- 角丸は自動適用されるため、四角形で作成
- 透過不可（背景色必須）
- シンプルで認識しやすいデザイン
- 小サイズでも識別可能

> 💡 **良いアイコンの特徴**
>
> ```
> 【成功するアイコンの条件】
>
> 1. シンプル（要素は1-2個まで）
>    ❌ 文字がごちゃごちゃ
>    ✅ シンボル1つ + 背景色
>
> 2. 認識しやすい（29pxでも識別可能）
>    → 通知バッジサイズでもわかる
>
> 3. 競合と差別化できる
>    → 同じカテゴリで並んだ時に目立つ
>
> 【避けるべきこと】
> ❌ 写真やリアルな画像
> ❌ 細かい文字（読めない）
> ❌ 角丸を自分で付ける（二重になる）
> ❌ 透明な背景
> ```

### 作成手順

1. 1024x1024 のマスターアイコンを作成
2. 各サイズにリサイズ
3. `assets/icons/` ディレクトリに保存

### Expo での設定

`app.json`:

```json
{
  "expo": {
    "icon": "./assets/icons/icon-1024.png",
    "ios": {
      "icon": "./assets/icons/icon-1024.png"
    }
  }
}
```

## 7.2 スクリーンショット

## 📚 学習ポイント：スクリーンショットの戦略

```
【スクリーンショットの役割】

1枚目: 「これ何ができるアプリ？」→ メイン機能を見せる
2枚目: 「具体的にどう使う？」→ 主要画面を見せる
3枚目: 「他にも機能ある？」→ サブ機能を見せる
4-5枚目: 「安心して使える？」→ 実績・特徴をアピール

【重要: 最初の2枚が勝負】
- 検索結果で見えるのは最初の2-3枚
- 詳細ページでも最初にスクロールなしで見えるのは2枚
- 1枚目で興味を引けなければ離脱

【効果的なスクリーンショットのパターン】

パターンA: シンプル画面
  [アプリ画面だけ]
  → 機能が一目瞭然な場合に有効

パターンB: デバイスフレーム + キャプション
  ┌─────────────────┐
  │ 「スキマ時間に」  │ ← キャッチコピー
  │  ┌──────────┐   │
  │  │ 画面    │   │ ← デバイスの中に画面
  │  └──────────┘   │
  │   iPhone風フレーム │
  └─────────────────┘
  → 何のアプリかを説明できる

パターンC: ストーリー形式
  1枚目→2枚目→3枚目 で使い方を順番に見せる
```

### 必要サイズ・枚数

| デバイス | サイズ (px) | 枚数 |
|---------|------------|------|
| iPhone 6.9" (15 Pro Max) | 1320x2868 | 5枚 |
| iPhone 6.7" (15 Plus) | 1290x2796 | 5枚 |
| iPhone 5.5" (8 Plus) | 1242x2208 | 5枚 |
| iPad 12.9" (Pro) | 2048x2732 | 5枚 |

### 必須画面

1. **ホーム画面（年度選択）**
   - 年度リストが見える状態
   - 進捗表示あり

2. **問題一覧画面**
   - 問題番号とカテゴリが見える
   - 解答済みマーク表示

3. **問題演習画面（解答前）**
   - 問題文と選択肢が見える

4. **問題演習画面（解答後）**
   - 正解/不正解のハイライト
   - 解説が表示された状態

5. **進捗・統計画面**
   - 学習進捗が可視化された状態


## 7.3 テキストコンテンツ

## 📚 学習ポイント：テキストのASO最適化

### 検索順位に影響する要素

```
【重要度ランキング】

1位: アプリ名（最大30文字）
   - 最も検索順位に影響
   - ブランド名 + キーワードが理想
   例: 「コンクリート診断士試験対策」

2位: サブタイトル（最大30文字）
   - アプリ名の補足
   - 追加のキーワードを含める
   例: 「スキマ時間で効率的に学習」

3位: キーワードフィールド（最大100文字）
   - ユーザーには見えない
   - 検索用のキーワードを羅列
   - カンマ区切り、スペースは無駄

【テクニック】
- アプリ名とサブタイトルに入れた単語は
  キーワードフィールドに入れる必要なし（重複）
- 単語を分けると組み合わせで検索される
  例:「診断士,試験」→「診断士試験」でもヒット
```

### 説明文の構成

```
【説明文の読まれ方】

┌─────────────────────────────┐
│ 最初の3行（展開前に見える）  │ ← 超重要！
├─────────────────────────────┤
│                             │
│ 「もっと見る」をタップで     │ ← ここは読まれにくい
│ 展開される部分               │
│                             │
└─────────────────────────────┘

【最初の3行に入れるべき内容】
1. このアプリで何ができるか
2. 誰のためのアプリか
3. 差別化ポイント

【残りの部分】
- 機能の詳細リスト
- 使い方
- 注意事項
```

### 説明文

```
コンクリート診断士試験の合格を目指す方のための問題演習アプリです。

【特徴】
・過去問100問を収録
・スキマ時間で手軽に学習
・正誤判定と詳しい解説
・学習進捗の可視化

【こんな方におすすめ】
・通勤時間を有効活用したい方
・効率的に試験対策を進めたい方
・自分の弱点を把握したい方

【機能】
・年度別問題演習
・選択式問題（4択）
・解答後の即時フィードバック
・カテゴリ別進捗管理
・学習履歴の記録

忙しい社会人でも、毎日のちょっとした時間でコツコツ学習できます。
コンクリート診断士試験合格に向けて、今日から始めましょう！

※本アプリの問題・解説は学習支援を目的としています。
```

**制限**: 4,000文字以内

```
コンクリート診断士,試験対策,過去問,問題集,学習アプリ,資格試験,土木,建築,診断士
```
